🧪 Experiment 10: Web Application using Docker

🎯 Objective

To learn Dockerfile instructions and build a Docker image for a sample web application.

📚 Theory

Dockerfile is a script containing instructions to build a Docker image. It defines the base image, working directory, dependencies, and execution command for an application. :contentReference[oaicite:1]{index=1}

🛠️ Tools Required

  • Docker
  • Python Flask
  • Linux / WSL

⚙️ Dockerfile Example

FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["python", "app.py"]

⚙️ Commands Used

1️⃣ Build Image

docker build -t flask-app .

2️⃣ Check Images

docker images

3️⃣ Run Container

docker run -d -p 5000:5000 flask-app

4️⃣ Verify Application

http://localhost:5000

📌 Procedure

  1. Create a sample Flask application
  2. Create requirements.txt file
  3. Write Dockerfile
  4. Build Docker image
  5. Run container
  6. Access application in browser

📊 Observations

  • Dockerfile automates image creation
  • Application runs inside container
  • Port mapping enables browser access
  • Environment remains consistent

💡 Conclusion

Docker simplifies application deployment by packaging code and dependencies together, ensuring consistent behavior across environments.
