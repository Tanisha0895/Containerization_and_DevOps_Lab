🧪 Experiment 12: Testing Containerized Application using Selenium

🎯 Objective

To develop automated test cases for a containerized web application using Selenium.

📚 Introduction

Selenium is a popular automation testing tool used for testing web applications. It simulates user interactions such as clicking buttons, entering data, and verifying outputs. This helps ensure that the application works correctly. :contentReference[oaicite:1]{index=1}

🛠️ Tools Required

  • Selenium WebDriver
  • Java / Python
  • ChromeDriver
  • Docker (for containerized app)

⚙️ Sample Test Code

from selenium import webdriver
from selenium.webdriver.common.by import By
import time

driver = webdriver.Chrome()

driver.get("http://localhost:5000")

# Verify page loads
heading = driver.find_element(By.ID, "greeting")
assert heading.text == "Hello, World!"

# Click button
driver.find_element(By.ID, "changeText").click()
time.sleep(1)

# Verify text change
heading = driver.find_element(By.ID, "greeting")
assert heading.text == "Hello, Selenium!"

driver.quit()

📌 Procedure

  1. Create containerized web application
  2. Install Selenium and WebDriver
  3. Write test script for application

📊 Test Cases

Test Case 1: Page Load

  • Open application
  • Check heading exists
  • Expected: "Hello, World!" displayed

Test Case 2: Button Click

  • Click button
  • Verify text change
  • Expected: "Hello, Selenium!"

📊 Observations

  • Automation testing reduces manual effort
  • Ensures application functionality
  • Test cases executed successfully

💡 Conclusion

Selenium helps automate testing of containerized applications, improving reliability, accuracy, and efficiency in DevOps workflows.
