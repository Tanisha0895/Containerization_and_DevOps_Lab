🧪 Experiment 3: Deploying NGINX using Different Base Images

🎯 Objective

To deploy NGINX using different base images (Official, Ubuntu, Alpine) and compare their size, performance, and layers.

🛠️ Prerequisites

  • Docker installed
  • Basic knowledge of Docker commands
  • Linux terminal

⚙️ Commands Used

1️⃣ Official NGINX Image

docker pull nginx
docker run -d --name nginx-official -p 8080:80 nginx
curl http://localhost:8080

2️⃣ Ubuntu-based NGINX Image

docker build -t nginx-ubuntu .
docker run -d --name nginx-ubuntu -p 8081:80 nginx-ubuntu
docker images nginx-ubuntu

3️⃣ Alpine-based NGINX Image

docker build -t nginx-alpine .
docker run -d --name nginx-alpine -p 8082:80 nginx-alpine
docker images nginx-alpine

4️⃣ Compare Image Size

docker images | grep nginx

5️⃣ Inspect Image Layers

docker history nginx-ubuntu
docker history nginx-alpine

📌 Procedure

  1. Pull official NGINX image
  2. Run container and verify output
  3. Create custom Dockerfile using Ubuntu
  4. Build and run Ubuntu-based image
  5. Create Dockerfile using Alpine
  6. Build and run Alpine-based image
  7. Compare size and layers of images

📊 Observations

  • Official NGINX image is optimized and medium-sized
  • Ubuntu-based image is large and has more layers
  • Alpine-based image is very small and lightweight
  • Alpine provides faster startup time

💡 Conclusion

Alpine-based images are best for lightweight and fast applications, while official images are suitable for production use. Ubuntu images provide more tools but increase size.
