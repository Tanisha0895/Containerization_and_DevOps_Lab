🧪 Experiment 4: Docker Essentials

🎯 Objective

To understand Dockerfile creation, building images, tagging, multi-stage builds, and publishing images to Docker Hub.

🛠️ Tools Required

  • Docker
  • Python / Node.js
  • Docker Hub account

⚙️ Commands Used

1️⃣ Build Docker Image

docker build -t my-flask-app .
docker images

2️⃣ Run Container

docker run -d -p 5000:5000 --name flask-container my-flask-app
docker ps

3️⃣ Tag Image

docker tag my-flask-app:latest username/my-flask-app:1.0

4️⃣ Push to Docker Hub

docker login
docker push username/my-flask-app:1.0

5️⃣ Multi-stage Build

docker build -f Dockerfile.multistage -t flask-multistage .
docker images

📌 Procedure

  1. Create a simple application (Flask / Node.js)
  2. Create a Dockerfile
  3. Build image using docker build
  4. Run container using docker run
  5. Tag image for Docker Hub
  6. Push image to Docker Hub
  7. Implement multi-stage build to reduce image size

📊 Observations

  • Dockerfile helps automate image creation
  • Tagging allows version control of images
  • Multi-stage builds reduce final image size
  • Publishing enables sharing images globally

💡 Conclusion

Dockerfile simplifies container creation, multi-stage builds optimize performance, and Docker Hub allows easy sharing and deployment of applications.
