🧪 Experiment 5: Docker Volumes, Environment Variables, Monitoring & Networks

🎯 Objective

To understand Docker volumes for persistent storage, environment variables for configuration, monitoring tools, and networking between containers.

🛠️ Tools Required

  • Docker
  • Linux / WSL
  • Terminal

📚 Theory

Docker containers are temporary (ephemeral), meaning data inside them is lost when the container is removed. Docker volumes solve this problem by storing data outside the container lifecycle. :contentReference[oaicite:0]{index=0} Docker networks allow communication between containers, enabling multi-container applications. :contentReference[oaicite:1]{index=1}

⚙️ Commands Used

1️⃣ Create and Use Volume

docker volume create mydata
docker run -d --name web1 -v mydata:/app/data nginx
docker volume ls
docker volume inspect mydata

2️⃣ Bind Mount

docker run -d --name web2 -v ~/mydata:/app/data nginx

3️⃣ Environment Variables

docker run -d -e APP_ENV=production -e DEBUG=false myapp
docker exec container-name printenv

4️⃣ Monitoring Commands

docker stats
docker logs container-name
docker top container-name
docker inspect container-name

5️⃣ Networking

docker network ls
docker network create app-network
docker run -d --name db --network app-network postgres
docker run -d --name web --network app-network myapp
docker network inspect app-network

📌 Procedure

  1. Create container and observe data loss
  2. Create Docker volume and mount it
  3. Use bind mounts for host mapping
  4. Set environment variables using -e and env files
  5. Monitor containers using Docker commands
  6. Create custom network and connect containers

📊 Observations

  • Volumes preserve data even after container removal
  • Environment variables allow dynamic configuration
  • docker stats shows real-time resource usage
  • Containers in same network communicate using names

💡 Conclusion

Docker volumes ensure persistent storage, environment variables enable flexible configuration, monitoring tools help track performance, and networks enable communication between containers, making Docker suitable for real-world applications.
