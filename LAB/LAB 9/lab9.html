🧪 Lab 9: Container Orchestration using Kubernetes

🎯 Objective

To automate the deployment and management of a containerized application using Kubernetes.

📚 Introduction

Kubernetes is an open-source container orchestration platform used to automate deployment, scaling, and management of containerized applications. It helps manage multiple containers efficiently across clusters. :contentReference[oaicite:1]{index=1}

🛠️ Tools Required

  • Docker
  • Minikube
  • Kubectl
  • Linux / WSL

⚙️ Commands Used

1️⃣ Start Minikube

minikube start

2️⃣ Use Minikube Docker Environment

eval $(minikube docker-env)
docker build -t myapp:1.0 .

3️⃣ Create Deployment

kubectl apply -f deployment.yaml
kubectl get pods

4️⃣ Expose Deployment

kubectl expose deployment myapp-deployment --type=NodePort --port=80
minikube service myapp-deployment

5️⃣ Update Deployment

docker build -t myapp:1.0 .
kubectl rollout restart deployment myapp-deployment
kubectl get pods

📌 Procedure

  1. Ensure Docker and Minikube are installed
  2. Start Minikube cluster
  3. Build Docker image
  4. Create deployment YAML file
  5. Apply deployment using kubectl
  6. Expose application using NodePort
  7. Access application in browser

📊 Observations

  • Kubernetes automatically manages container replicas
  • Pods are created and managed dynamically
  • Application can be scaled easily
  • Service provides stable access to application

💡 Conclusion

Kubernetes simplifies container orchestration by automating deployment, scaling, and management of applications, making it suitable for production environments.
