01 — Overview

What this project does

🐍
Flask Backend
Python Flask server serving a web page and a REST API endpoint on port 5000.
🐘
PostgreSQL DB
Relational database with auto-initialized schema via init.sql on container startup.
🌐
IPvlan Network
Custom Docker IPvlan network giving each container its own static IP on the LAN.
💾
Named Volume
Docker named volume pgdata ensures data persists across container restarts.

version: "3.9"

services:

  backend:
    build: ./Backend
    container_name: pa1_backend
    ports:
      - "5000:5000"
    environment:
      DB_HOST: db
      POSTGRES_USER: myuser
      POSTGRES_PASSWORD: mypassword
      POSTGRES_DB: mydb
    depends_on:
      - db
    networks:
      ipvlan_net:
        ipv4_address: 192.168.170.10

  db:
    image: postgres:15-alpine
    container_name: pa1_db
    environment:
      POSTGRES_USER: myuser
      POSTGRES_PASSWORD: mypassword
      POSTGRES_DB: mydb
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./database/init.sql:/docker-entrypoint-initdb.d/init.sql
    networks:
      ipvlan_net:
        ipv4_address: 192.168.170.11

networks:

04 — Setup

How to run this project

01
Create Project Directory
Create and navigate into the project folder in WSL2 terminal.
bash
mkdir devops-project
cd devops-project
02
Create IPvlan Network (run only once)
Creates a custom Docker IPvlan network with static IPs for containers.
bash
docker network create \
  -d ipvlan \
  --subnet=192.168.160.0/20 \
  --gateway=192.168.160.1 \
  -o ipvlan_mode=l2 \
  -o parent=eth0 \
  ipvlan_net
03
Build & Start Containers
Docker Compose builds the Flask image and starts both containers in detached mode.
bash
docker compose up --build -d
04
Verify Everything is Running
Check containers, network, images and volumes.
bash
docker ps
docker network inspect ipvlan_net
docker volume ls
docker images
05
Open App in Browser
Visit the running Flask application and test the API endpoint.
bash
# Open web app
http://localhost:5000

# Test API
curl http://localhost:5000/api

06 — Volume Persistence

Data survives container restarts

The named volume pgdata stores PostgreSQL data permanently. Even after stopping all containers, data remains intact.

Start Containers

Run docker compose up -d. Both Flask and PostgreSQL start with pgdata volume mounted.

Stop Containers

Run docker compose down. Containers stop but the named volume is NOT deleted.

Restart & Verify

Run docker compose up -d again. Volume and data still present! ✓

bash — persistence test
# 1. Start containers
docker compose up -d

# 2. Stop containers (volume stays!)
docker compose down

# 3. Confirm volume still exists
docker volume ls | grep pgdata

# 4. Restart
docker compose up -d

# 5. Volume is still there ✓
docker volume ls
